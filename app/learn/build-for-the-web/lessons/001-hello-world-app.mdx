## Step 1: Make something appear üß™

- The objective of this step is simple:

  > Make sure your code shows up in the browser.

- ‚ö†Ô∏è **Note**
  - This is the **only time** we'll use **red text** in this lesson.
  - It exists for one reason:
    > To clearly confirm that Tailwind CSS is working
- ‚úèÔ∏è Replace the contents of home.tsx with:

```tsx
export default function Home() {
	return (
		<h1 className='m-4 text-4xl font-bold text-red-600'>
			Hello, World!
		</h1>
	)
}
```

### What this file represents

- `home.tsx` is a **route file**.
- In React Router Framework Mode:
  - A file named home.tsx.
  - Automatically becomes the home page.
- You didn't configure routing.
- You didn‚Äôt wire anything manually.
- The framework handled it.

### Line-by-line explanation üß†

#### 1Ô∏è‚É£ `export default`

```ts
export default function Home() {
```

- React Router expects **one default export** per route file.
- That export is what gets rendered on the page.

#### 2Ô∏è‚É£ `function Home()`

- This is a React **component**.
- A component is just a function.
- Its only job:
  > Return UI.

#### 3Ô∏è‚É£ `return (...)`

- Whatever is returned here:
  - Becomes visible in the browser
- No return ‚Üí no UI

#### 4Ô∏è‚É£ JSX

```tsx
<h1 className='text-4xl font-bold text-red-600 m-4'>
```

- This looks like HTML.
- It's actually JSX.
- JSX lets you write UI inside JavaScript.

- Important detail:
  - We use `className`, not `class`.

#### 5Ô∏è‚É£ Tailwind classes

- `text-4xl` ‚Üí large text.
- `font-bold` ‚Üí strong emphasis.
- `text-red-600` ‚Üí visible confirmation.
- `m-4` ‚Üí spacing around the element.

These styles apply **only** to this element.
