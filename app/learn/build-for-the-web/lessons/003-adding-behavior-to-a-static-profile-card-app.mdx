- This lesson intentionally does **not** introduce:

  - Animations
- Those are important ‚Äî but not yet.

## Mental Model üß†

- Before touching code, understand this idea:
  > **UI = a function of state**
  >
  > - Same state ‚Üí same UI
  > - Different state ‚Üí different UI
- This means:
  - You don't manually update the screen.
  - You don't change text directly.
  - You change **state**.
  - React re-runs your component.
  - The UI updates automatically.
- In Lesson 2, your UI had **no state**.
- It always rendered the same output.
- In this lesson, the UI responds to **what's happening**.

<hr className='my-8' />

## Starting Point üèÅ

- Open the project from **Lesson 2**.
- You should already have a fully styled Profile Card.
- The **Follow** button:
  - Looks clickable
  - Has hover feedback
  - But does nothing when clicked

## Step 1: Identify What Should Change üß©

- Ask one simple question:
  > What is the _smallest_ piece of information that affects this UI?
- Answer:
  - Whether the user is following or not.
- Nothing else needs to change.
- We'll represent this with a boolean:
- `false` ‚Üí not following.
- `true` ‚Üí following.

<hr className='my-8' />

## Step 2: Introduce State ‚öôÔ∏è

- At the top of your `home.tsx` file, import `useState`:

```ts
import { useState } from 'react'
```

- This line gives us access to React's state feature.
- We'll focus on _what it does_, not how imports work yet.
- Then, **inside** the `Home` component, add:

```ts
const [isFollowing, setIsFollowing] = useState(false)
```

- _What this line means_
  - `isFollowing`
    ‚Üí the current state value.
  - `setIsFollowing`
    ‚Üí a function that updates the state.
  - `false`
    ‚Üí the starting value
- That's all state is.
- A value ‚Äî and a way to update it.

<hr className='my-8' />

## Step 3: Connect User Action to State üñ±Ô∏è

- Now we connect a **click** to a **state change**.
- Update the **Follow** button:

```tsx
<button
	type='button'
	onClick={() => setIsFollowing(!isFollowing)}
	className='flex-1 rounded-lg bg-blue-600 px-4 py-2 text-sm text-white transition hover:bg-blue-700'
>
	Follow
</button>
```

- **What‚Äôs happening here**
  - The button listens for a click using `onClick`.
  - When clicked, we update state.
  - Instead of setting a fixed value, we **flip** the current state.
- This line is the key:

```ts
setIsFollowing(!isFollowing)
```

- It means:
  - If `isFollowing` is `false`, it becomes `true`.
  - If `isFollowing` is `true`, it becomes `false`.

- The button now represents a **real interaction**, not a one-time action.

<hr className='my-8' />

#### Step 4: Make the UI Respond to State üîÅ

- Now that the state can change in both directions, the UI should reflect it.
- Replace the button text with a conditional expression:

```tsx
{isFollowing ? 'Following' : 'Follow'}
```

- Your button now becomes:

```tsx
<button
	type='button'
	onClick={() => setIsFollowing(!isFollowing)}
	className='flex-1 rounded-lg bg-blue-600 px-4 py-2 text-sm text-white transition hover:bg-blue-700'
>
	{isFollowing ? 'Following' : 'Follow'}
</button>
```

- **What you should observe üëÄ**
  - First click ‚Üí Follow ‚Üí Following
  - Second click ‚Üí Following ‚Üí Follow
  - Every click updates the UI.
  - You didn't manually change the text.
  - You changed **state** ‚Äî and React re-rendered the UI.

<hr className='my-8' />

## Key Idea Reinforced üß†

> **UI is a function of current state**.

- Events update state.
- State changes trigger re-renders.
- The UI always reflects the latest state.
- This is the foundation of real interactivity.
