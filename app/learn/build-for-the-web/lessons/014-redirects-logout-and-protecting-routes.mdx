## Introduction 🧭

- In the previous lesson, something fundamental changed.
  - The system gained **identity at runtime**.

  ```text
  Request → Session → userId
  ```

  - The system can now answer:

  > **Who is making this request right now?**
  - That is a major step.

- But something is still missing.

#### The Problem

- The system knows:

  > Who is using the system

- But it does not decide:
  > What should happen next

#### What this looks like

- After login:

  ```text
  Session created → stay on /login
  ```

- If not logged in:

  ```text
  Request → no userId → error
  ```

- There is no:
  - Redirection
  - Logout
  - Flow control

#### The Real Question

- A real system must decide:

> What should happen after identity is known?

- Not sometimes
- Not manually
- But **on every request**

#### What This Lesson Introduces

- By the end of this lesson, you will:
  - Redirect users after login
  - Prevent access to invalid routes
  - Add logout functionality
  - Control application flow

> This lesson is about **flow at runtime**

## Step 2: Redirect After Login 🔁

- Open:

```text
app/data/auth.server.ts
```

- Your current `login` function should look like this:

```ts
export async function login(request: Request, userId: string) {
	const session = await getSession(request.headers.get('Cookie'))

	session.set('userId', userId)

	return new Response(null, {
		headers: {
			'Set-Cookie': await commitSession(session),
		},
	})
}
```

- Right now:
  - The session is created
  - The cookie is sent
  - But nothing tells the browser where to go next

```text
Login → session created → stay on current page
```

#### Replace the return statement

- Inside the same `login` function:
  - Find:

  ```ts
  return new Response(null, {
    headers: {
      'Set-Cookie': await commitSession(session),
    },
  })
  ```

  - Replace with:

  ```ts
  return redirect('/', {
    headers: {
      'Set-Cookie': await commitSession(session),
    },
  })
  ```

  - Also add this import at the top of the file

  ```ts
  import { redirect } from 'react-router'
  ```

  - Your updated file should now look like this

  ```ts
  import { redirect } from 'react-router'
  import { getSession, commitSession } from './session.server'

  export async function login(request: Request, userId: string) {
    const session = await getSession(request.headers.get('Cookie'))

    session.set('userId', userId)

    return redirect('/', {
      headers: {
        'Set-Cookie': await commitSession(session),
      },
    })
  }
  ```

#### What just happened?

- Before:

```text
Login → session created → stay on current page
```

- After:

```text
Login → session created → redirect → /
```

#### Why this matters

- Authentication now completes with navigation.

> - The system not only creates identity
> - It also moves the user forward

## Step 3: Protect the Login Route 🚧

- There is another problem.
- Right now:

```text
User logs in → session exists → /login still accessible
```

- That means:

```text
Authenticated user → can still visit login page
```

- That is incorrect.

#### Why this matters

- Once a user is logged in:

```text
/login no longer makes sense
```

- The system already knows who the user is.

> Logged-in users should not see login again

#### Reality

```text
Logged-in user → /login → redirect → /
```

- The login page should only exist for users who are **not authenticated**.

### Step 3a: Add a loader to the login route

- Open:

```bash
app/routes/login.tsx
```

- Your file currently contains:

```ts
import { Form } from 'react-router'
import type { Route } from './+types/login'
import { login } from '~/data/auth.server'

export async function action({ request }: Route.ActionArgs) {
	return login(request, 'user-1')
}
```

- This route can create a session.
- But it does not yet check whether a session already exists.

#### Add a loader above the action

- Add the following import:

```ts
import { redirect } from 'react-router'
import { getCurrentUserId } from '~/data/auth.server'
```

- Then add this loader **above** your `action` function:

```ts
import { Form } from 'react-router'
import { redirect } from 'react-router'

import type { Route } from './+types/login'

import { login } from '~/data/auth.server'
import { getCurrentUserId } from '~/data/auth.server'

export async function loader({ request }: Route.LoaderArgs) {
	const userId = await getCurrentUserId(request)

	if (!userId) {
		return null
	}

	return redirect('/')
}

export async function action({ request }: Route.ActionArgs) {
	return login(request, 'user-1')
}
```

#### What this does

- When someone visits `/login`:

```text
Request → check session → check userId
```

- If no user exists:

```text
No session → allow login page
```

- If a user already exists:

```text
Session exists → redirect → /
```

#### What just happened?

- Before:

```text
Authenticated user → /login → allowed
```

- After:

```text
Authenticated user → /login → redirect
```

#### Key Insight

> - Routes should not only render
> - They should also decide access based on identity

## Step 4: Introduce Logout 🚪

- The system can now:
  - Create a session
  - Identify the current user
  - Keep the user authenticated across requests
- But something is still missing.

#### The Problem

- Once a session is created:

```text
Login → session created → user stays authenticated
```

- That session remains active.
- Right now, there is no way to:

```text
Remove the session
```

- The system can log a user in.
- But it cannot log a user out.

#### Why this matters

- Authentication should not be permanent.
- Users should be able to:
  - Leave the system
  - End their session
  - Remove identity from the browser

> Identity must be removable

#### Reality

```tsx
User → logout → session removed
```

- The system must support both:

```text
Login → create identity
Logout → remove identity
```

### Step 4a: Create a Logout Function

- Open:

```bash
app/data/auth.server.ts
```

- Your file currently looks similar to this:

```ts
import { redirect } from 'react-router'
import { getSession, commitSession } from './session.server'

export async function login(request: Request, userId: string) {
	const session = await getSession(request.headers.get('Cookie'))

	session.set('userId', userId)

	return redirect('/', {
		headers: {
			'Set-Cookie': await commitSession(session),
		},
	})
}

export async function getCurrentUserId(request: Request) {
	const session = await getSession(request.headers.get('Cookie'))

	return session.get('userId')
}

export async function requireUserId(request: Request) {
	const userId = await getCurrentUserId(request)

	if (!userId) {
		throw new Response('Unauthorized', { status: 401 })
	}

	return userId
}
```

- Right now:
  - You can create a session
  - But you cannot destroy one

#### Update the import

- Find:

```ts
import { getSession, commitSession } from './session.server'
```

- Replace with:

```ts
import { getSession, commitSession, destroySession } from './session.server'
```

#### Add the logout function

- Below `requireUserId`, add:

```ts
export async function logout(request: Request) {
	const session = await getSession(request.headers.get('Cookie'))

	return redirect('/login', {
		headers: {
			'Set-Cookie': await destroySession(session),
		},
	})
}
```

#### Your file should now include

```ts
import { redirect } from 'react-router'
import { getSession, commitSession, destroySession } from './session.server'

export async function login(request: Request, userId: string) {
	const session = await getSession(request.headers.get('Cookie'))

	session.set('userId', userId)

	return redirect('/', {
		headers: {
			'Set-Cookie': await commitSession(session),
		},
	})
}

export async function logout(request: Request) {
	const session = await getSession(request.headers.get('Cookie'))

	return redirect('/login', {
		headers: {
			'Set-Cookie': await destroySession(session)
		}
	})
}

export async function getCurrentUserId(request: Request) {
	const session = await getSession(request.headers.get('Cookie'))

	return session.get('userId')
}

export async function requireUserId(request: Request) {
	const userId = await getCurrentUserId(request)

	if (!userId) {
		throw new Response('Unauthorized', { status: 401 })
	}

	return userId
}
```

#### What this does

- Reads the current session
- Destroys the session
- Removes the cookie from the browser
- Redirects the user to `/login`

#### What just happened?

- Before:

```text
Session created → identity persists
```

- After:

```text
Session destroyed → identity removed
```

#### Why this matters

- Authentication is not only about entering
- It is also about leaving

## Step 8: The Full System Flow 🔄

- At this point:
  - Users can log in
  - Sessions can be created
  - Sessions can be destroyed
  - Protected routes can redirect
  - Identity now affects behavior

> - The system no longer only knows the user
> - It now responds to the user

#### Login Flow

- A user visits:

```text
/login
```

- Then submits the login form.
- The system performs:

  ```text
  Request → login() → session created → redirect → /
  ```

- This means:
  - A session is stored
  - A cookie is sent to the browser
  - The user becomes authenticated

#### Authenticated Request

- When a user visits a protected route:

  ```text
  Request → requireUserId(request)
  ```

- The system checks:

  ```text
  Does this request contain identity?
  ```

- If a user exists:

  ```text
  Request → Session → userId → Response
  ```

- The request continues normally.

#### Logout Flow

- When the user clicks Logout:

```text
POST /logout
```

- The system performs:

```text
Request → logout() → destroySession() → redirect → /login
```

- This means:
  - The session cookie is removed
  - Identity is removed
  - The user leaves the authenticated state

#### Unauthorized Request

- If a request reaches a protected route:

```text
No session → requireUserId()
```

- The system responds with:

  ```text
  Redirect → /login
  ```

- The request does not continue.

#### What just happened?

- Before:

  ```text
  Identity existed
  ```

- After:

  ```text
  Identity controls system behavior
  ```

> Every request now produces a decision
